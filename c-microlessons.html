The CS50 Plurality problem requires the implementation of a program that simulates a plurality election, also known as "first-past-the-post" or "winner take all". In this electoral system, each voter casts a single vote for one candidate. Upon the conclusion of the election, the candidate who has accumulated the greatest number of votes is declared the winner. The program's operational flow involves several stages: it accepts candidate names as command-line arguments, prompts the user for the total number of voters, systematically collects each vote, and ultimately identifies and prints the winner or winners of the election.

The plurality.c file, provided as part of the assignment, includes several predefined components crucial for the election simulation. A global array of these candidate structures, named candidates, is also provided, serving as the primary repository for all participating candidates in the election.

The assignment specifically tasks the implementer with completing two functions: vote and print_winner. The vote function is designed to accept a single argument, string name, which represents the name of the candidate for whom a vote is cast. The function's responsibility is to search for a matching candidate within the candidates array. If a match is found, the corresponding candidate's vote total must be incremented, and the function should return true to signify a successful ballot. Conversely, if the provided name does not correspond to any candidate in the election, no vote totals should be altered, and the function must return false to indicate an invalid ballot. The print_winner function is charged with identifying the candidate or candidates who have received the highest number of votes. In scenarios where multiple candidates achieve the maximum vote count (a tie), the function must print the names of all such winning candidates, each on a separate line. A critical constraint of this assignment is that only the implementations of the vote and print_winner functions are to be modified; no other parts of plurality.c should be altered, aside from potentially including additional header files. Micro Lesson 1: Loops in C - Mastering Repetition

Loops are fundamental control structures in C that enable a block of code to be executed repeatedly.[2][3] This automation is critical for tasks involving the processing of data collections, eliminating the need for redundant code, and ensuring programs are scalable.[2] C offers three primary types of loops: for, while, and do-while, each designed for distinct scenarios.[2][3]

The user's challenge with maintaining pace and solving programming problems often stems from a difficulty in efficiently handling repetitive operations. Loops directly address this by serving as the engine for data processing. The Plurality assignment, for instance, requires processing a variable candidate_count of candidates and a number of voters. Manually writing code for each candidate or voter would be impractical and inefficient. Loops provide the necessary automation and scalability to handle varying numbers of inputs.[2] For example, a common error observed in attempts to implement the vote() function is that the loop runs only once due to incorrect logic.[4] This highlights that loops are not merely a syntactic feature but a core algorithmic necessity for correct and efficient problem-solving. A misunderstanding of loop mechanics, such as premature returns, can derail the program's logic entirely.

Understanding for Loops

The for loop is typically employed when the number of iterations is known beforehand.[2][3] Its syntax integrates initialization, condition-checking, and increment/decrement into a single line, making it concise and suitable for tasks like counting or iterating over arrays.[2]

The basic syntax for a for loop is: for (initializationStatement; testExpression; updateStatement) { // statements inside the body of loop }.[8][9][10]

  • Initialization: This step is executed only once at the very beginning of the loop. It is typically used to declare and initialize a loop control variable, for instance, int i = 0.[8][10]
  • Condition: This is a logical expression that is evaluated before each iteration. If the testExpression evaluates to true (non-zero), the loop body executes. If it evaluates to false (zero), the loop terminates.[8][10]
  • Increment/Decrement: This statement updates the loop control variable after each iteration of the loop body. Its purpose is to ensure that the loop progresses towards its termination condition.[8][10]

The for loop operates in a sequential manner: first, the initialization statement is executed once. Then, the testExpression is evaluated. If true, the statements within the loop body are executed, followed by the execution of the updateStatement. This cycle of condition evaluation, body execution, and update continues until the testExpression becomes false.[9][10]

A common practical application of the for loop is iterating through an array. For example, to print elements of an integer array:

#include <stdio.h> int main() { int arr[] = {10, 20, 30, 40, 50}; // Corrected array declaration for (int i = 0; i < 5; i++) { printf("%d ", arr[i]); } printf("\n"); // Added for better output formatting return 0; } // Output: 10 20 30 40 50

In this example, the loop variable i serves as an index to access each element of the array sequentially.[8][11] This direct application is highly relevant to processing the candidates array in the Plurality assignment.

Understanding while Loops

The while loop is used when the exact number of iterations is not known beforehand and depends on a condition being true.[2][3] It is an entry-controlled loop, meaning the condition is checked *before* the loop body executes.[2][3][12] If the condition is initially false, the loop body will not execute even once.[12]

The basic syntax for a while loop is: while (condition) { // body; updation; }.[2][12] Initialization of the loop variable typically occurs *before* the while loop, and the update statement must be explicitly included *within* the loop body to ensure the condition eventually becomes false and the loop terminates.[12]

The while loop operates by first evaluating its condition. If the condition is true, the loop body executes. After the body executes, program control returns to the start, and the condition is re-evaluated. This process continues as long as the condition remains true.[12] while loops are ideal for scenarios where the loop needs to continue until a certain external condition is met, such as continually reading user input until a valid number is entered.[2]

#include <stdio.h> int main() { int i = 1; while (i <= 5) { printf("GfG\n"); i++; } return 0; } // Output: GfG GfG GfG GfG GfG

This example demonstrates how the loop continues as long as i is less than or equal to 5, printing "GfG" in each iteration and incrementing i.[12]

do-while Loops: The "At Least Once" Guarantee

The do-while loop is similar to a while loop, but its key distinguishing feature is that the loop body executes *at least once* before the condition is checked.[2][13] This makes it an exit-controlled or post-tested loop.[2][13] Even if the condition is initially false, the code inside the do block will execute one time.[13]

The syntax for a do-while loop is: do { // body; updation; } while (condition);.[2][13] It is important to note the semicolon at the end of the while statement, which is unique to do-while loops.[13]

The do-while loop first executes the code block within the do part. After this initial execution, the condition in the while part is evaluated. If the condition is true, the loop repeats. This process continues until the condition evaluates to false, at which point the loop terminates.[13] This type of loop is particularly useful for scenarios where an action must occur at least once, such as displaying a menu to a user or ensuring initial input is received before validation.[2]

#include <stdio.h> #include <stdbool.h> int main() { bool c = false; do { printf("This is loop body.\n"); } while (c); return 0; } // Output: This is loop body.

Even though c is initially false, the printf statement inside the do block executes once before the condition is checked and the loop terminates.[13]

Nested Loops

Nested loops involve placing one loop inside another. The inner loop executes completely for every single iteration of the outer loop.[2][3][10] This structure is powerful for handling multi-dimensional data or tasks that require layered repetition.

This concept is vital for the print_winner function in Plurality. As suggested by external resources, it is often necessary to use two separate passes (which can be implemented with two loops) to determine the winner(s).[4] The first loop can be used to find the maximum vote count among all candidates. Once the maximum is known, a second loop can iterate through the candidates again to print the names of all candidates who achieved that maximum score, thereby correctly handling potential ties.[4]

While for loops are frequently chosen for iterating through known collections like the candidates array, understanding while and do-while loops is crucial for designing robust programs that can handle diverse input scenarios.[2][13] For instance, if the Plurality assignment required repeatedly asking for user input *until* a valid vote was entered, a do-while loop might be more appropriate than a for loop, as it guarantees the prompt appears at least once. This understanding helps a programmer move beyond rote memorization of syntax to a deeper comprehension of *when* to apply each tool, fostering better design choices and more resilient code. A learner's struggle may indicate they are applying the wrong loop type or mismanaging loop conditions, leading to incorrect or inefficient code, and grasping the nuances of each loop type is key to overcoming this.

Micro Lesson 2: Structures in C - Organizing Your Data

A struct (short for structure) in C is a powerful user-defined data type that enables the grouping of variables of *different* data types under a single, meaningful name.[15][16] This contrasts with arrays, which are collections of elements of the *same* data type.[11][15] Structures are fundamental for modeling complex, real-world entities within a program, such as a candidate in an election.

The general syntax for defining a struct is:

struct structure_name { data_type1 member1; data_type2 member2; //... more members }; // Don't forget the semicolon!

It is crucial to understand that defining a struct does not allocate any memory; it merely creates a blueprint or template for a new data type.[16] Memory is only allocated when a variable of that struct type is declared.

To declare a variable of a defined struct type, the syntax is: struct structure_name var_name;.[16] Variables can also be declared directly after the struct definition.[16]

To interact with individual data items (members) within a declared struct variable, the . (dot) operator is used.[15][16][17] The syntax for accessing a member is struct_variable.member_name;.[16] For instance, if a struct Player is defined with name and score members, player1.name and player1.score would be used to access or modify those specific members for a player1 variable.[15]

In plurality.c: struct candidate { string name; int votes; };. If alice is a candidate, alice.name and alice.votes access her data.

#include <stdio.h> #include <string.h> // Required for strcpy // It's good practice to define structs globally or in a header if used by multiple functions struct Player { char name[50]; // Character array for name int score; }; int main() { struct Player player1; // Declare a Player variable // Initialize members strcpy(player1.name, "Alice"); // Use strcpy for string assignment player1.score = 100; printf("Player Name: %s\n", player1.name); printf("Player Score: %d\n", player1.score); return 0; } // Output: // Player Name: Alice // Player Score: 100

The ability to organize related, disparate data types under a single, meaningful name (candidate) is a crucial step in translating real-world problems into structured code. This fosters better data organization and makes the code more readable and maintainable. Without structures, one might attempt to manage candidate names in one array and votes in another, relying on parallel indices, which is prone to errors and less intuitive.

While conceptually straightforward, the . (dot) operator is the critical syntactic mechanism for interacting with the data encapsulated within a struct.[16] The ability to correctly access and increment candidates[i].votes is fundamental for the vote() function.[4] Without a clear understanding of how to use the dot operator, a programmer cannot access or modify the name or votes members of a candidate struct, directly impeding the implementation of the core logic for vote and print_winner. This highlights that even seemingly small syntactic details are crucial for successful program implementation.

Micro Lesson 3: Arrays of Structures - Managing Collections

While a single struct variable holds data for a *single* entity, an array of structures enables the management of *multiple* entities that share the same underlying structure.[11][18] This can be conceptualized as a list or table where each row represents a struct instance, such as a list of students or, in the context of Plurality, a list of candidates.[18]

The syntax for declaring an array of structs is: struct StructName arrayName;.[11][18] For example, struct Student students; declares an array named students where each element students[i] is a Student struct.[18]

Arrays of structs can be initialized in several ways:

  • Static Initialization (at Declaration): Values can be assigned directly when the array is declared using nested curly braces, where each inner brace corresponds to a struct element.[18][19]
#include <stdio.h> struct Student { int rollNo; char name[50]; // Corrected to char array for string float marks; }; int main() { struct Student students[] = { // Corrected to array initialization {101, "Rahul", 89.5}, {102, "Sneha", 92.0}, {103, "Aman", 78.5} }; printf("Name: %s, Marks: %.2f\n", students[1].name, students[1].marks); // Accessing an element return 0; } // Output: Name: Sneha, Marks: 92.00
  • Dynamic Initialization (Using User Input/Loops): In many practical scenarios, such as the Plurality assignment where candidate names are provided as command-line arguments and votes are user input, the array of structs is populated during program execution using loops and input functions like scanf.[11][18]

A critical consideration when working with char arrays (strings) within structs is that direct assignment using the = operator is not permissible *after* the struct variable has been declared.[18][19] Instead, the strcpy() function (or strncpy() for safer string handling) must be employed. For example: strcpy(students[i].name, "Alice");.

To efficiently process data within an array of structs, combining loops with the dot operator is the standard approach.[11][18] The syntax for accessing a member within an array of structs is arrayName[index].member_name;. A for loop can iterate through an array of Student structs to print each student's details:

#include <stdio.h> #include <string.h> // For strcpy struct Student { char name[50]; int roll; float marks; }; int main() { struct Student students[3]; // Array of 3 Student structs // Initialize the first student strcpy(students[0].name, "John"); students[0].roll = 1; students[0].marks = 85.5; // Initialize the second student strcpy(students[1].name, "Emma"); students[1].roll = 2; students[1].marks = 90.6; // Initialize the third student strcpy(students[2].name, "Harry"); students[2].roll = 3; students[2].marks = 92.7; printf("Students:\n"); for (int i = 0; i < 3; i++) { printf("Name: %s, Roll: %d, Marks: %.2f\n", students[i].name, students[i].roll, students[i].marks); } return 0; } /* Output: Students: Name: John, Roll: 1, Marks: 85.50 Name: Emma, Roll: 2, Marks: 90.60 Name: Harry, Roll: 3, Marks: 92.70 */

This demonstrates how the loop variable i is used as an index to access each Student struct within the students array, and then the dot operator is used to access individual members like name, roll, and marks for each student.[11][18]

In the context of Plurality, the global array of candidates is a direct application of an array of structures. The vote function will utilize a loop to iterate through this candidates array to find a matching name, while the print_winner function will employ loops to determine the maximum vote count and then identify all candidates who achieved that maximum within this array.

The need to manage multiple candidates in the Plurality election makes arrays of structs the only practical and scalable solution for handling a collection of candidates up to MAX.[1][11][18] If individual struct variables were used (e.g., candidate1, candidate2), the code would become unmanageable and unscalable as the number of candidates increases. An array of structs provides a single, organized variable to hold all candidate data, making it straightforward to access them via an index. This is a crucial step in building programs that can handle varying amounts of input data efficiently, a core principle of good software design.

The most common and essential operation on an array of structs is iterating through it with a loop to access and manipulate individual struct members.[11][18] This represents a powerful synergy between "Loops" and "Structures." The vote function needs to check every candidate's name to find a match, and the print_winner function needs to examine every candidate's vote count. An array of structs provides the structured collection of data, and loops provide the efficient mechanism to traverse and process each element within that collection. Without this synergy, processing the election data would be impossible or incredibly cumbersome. The external guidance for vote() about needing to test "each candidate name" [4] directly implies the necessity of this loop-struct synergy for correct implementation.

Micro Lesson 4: Conditional Logic - Making Decisions

Conditional statements are fundamental control structures that empower a program to make decisions, executing one block of code if a specified condition is true, and a different block if the condition is false.[6] This capability allows programs to respond dynamically to varying inputs or states.

if-else Statements for Binary Choices

The if-else statement is used for binary choices, where a program needs to choose between two possible paths of execution.

The syntax for if-else is:

if (condition) { // Code to execute if condition is true } else { // Code to execute if condition is false }

The condition is evaluated first. If it is non-zero or non-null, it is considered true, and the code within the if block is executed, while the else block is skipped. If the condition evaluates to zero or null (considered false), the code within the else block is executed.[6]

For example, to check if a number is greater than 5:

#include <stdio.h> int main() { int n = 10; if (n > 5) { printf("%d is greater than 5\n", n); } else { printf("%d is less than or equal to 5\n", n); } return 0; } // Output: 10 is greater than 5

Here, since n (10) is indeed greater than 5, the if block executes.[6]

if-else if Ladders for Multiple Conditions

When a program needs to check multiple conditions sequentially, the if-else if ladder is employed. This structure extends the basic if-else by allowing a series of conditions to be tested. The program proceeds down the "ladder," checking each condition in order. The first condition that evaluates to true will have its corresponding block of code executed, after which the entire if-else if structure is exited. If none of the if or else if conditions are true, the final else block (if present) is executed.

The syntax for an if-else if ladder is:

if (condition1) { // Code if condition1 is true } else if (condition2) { // Code if condition2 is true } else if (condition3) { // Code if condition3 is true } else { // Code if all conditions are false }

Only one block of code within the entire ladder will execute.

An example of evaluating a number against multiple specific values:

#include <stdio.h> int main(){ int number = 5; if (number == 20) { printf("number is equal to 20\n"); } else if (number == 50) { printf("number is equal to 50\n"); } else if (number == 100) { printf("number is equal to 100\n"); } else { printf("number is not equal to 20, 50 or 100\n"); } return 0; } // Output: number is not equal to 20, 50 or 100

In this case, since number is 5, none of the if or else if conditions are met, leading to the execution of the final else block.

Using Logical Operators (&&, ||) for Complex Conditions

Logical operators allow the combination of multiple conditions within a single if or else if statement, enabling more sophisticated decision-making logic.[5][6]

  • && (Logical AND): The combined condition is true only if both conditions on its left and right sides are true.
  • || (Logical OR): The combined condition is true if at least one of the conditions on its left or right side is true.

For example, to check if a number is both positive AND even:

#include <stdio.h> int main() { int No = 10; if (No > 0 && No % 2 == 0) { printf("Number is positive and even\n"); } else if (No > 0 && No % 2!= 0) { printf("Number is positive and odd\n"); } else if (No < 0) { printf("Number is negative\n"); } else { printf("Number is zero\n"); } return 0; } // Output: Number is positive and even

Here, No is 10, which is greater than 0 AND is even, so the first if condition is true, and its block executes.

Conditional statements serve as the program's decision-making brain. The ability to translate real-world decisions into code is a common hurdle in programming. In Plurality, the program must decide if a vote is valid and which candidate(s) win. These are explicit decision points. Without a strong grasp of conditional logic, the program cannot adapt to different inputs (e.g., a valid versus an invalid vote) or different scenarios (e.g., a single winner versus a tie). This makes conditionals not just a syntactic feature but the very core of a program's ability to solve problems dynamically. An observation from external resources regarding the vote() function's if..else returning too early [4] directly points to a misunderstanding of how conditional flow should be structured within a loop to achieve the desired outcome. This highlights that correct conditional logic is paramount for accurate program behavior.

A subtle but critical point for the vote function involves comparing candidate names. Unlike integers, strings in C are character arrays and *cannot* be compared directly using the == operator for equality. Using == with character arrays compares their memory addresses, not their content, which is a common beginner error. Instead, the strcmp() function (string compare) is required for this purpose. strcmp() returns 0 if the strings are identical.[4] Explicitly understanding the need for strcmp() and why direct == does not work for strings is a vital point that prevents a common, frustrating bug that can be difficult for a beginner to diagnose, thereby streamlining the problem-solving process for Plurality.

Micro Lesson 5: Functions in C - Modular Solutions

A function in C is a self-contained block of statements designed to perform a specific, well-defined task when called.[7] Functions are fundamental building blocks for organizing C programs and offer significant benefits:

  • Modularity: They enable the breakdown of a large, complex problem (like an election simulation) into smaller, more manageable, and easier-to-understand sub-problems. This reduces cognitive load and enhances development efficiency.[7]
  • Reusability: A piece of code, once written as a function (e.g., a function to calculate a square [20]), can be called multiple times from different parts of the program without needing to be rewritten.[7]
  • Readability and Maintainability: Well-named functions make code easier to read, understand, and debug. Changes to a specific part of the logic can often be isolated within a single function.[7]

Defining, Calling, and Understanding Function Parameters and Return Types

Function Definition: This involves specifying the function's return_type, its function_name, and a parameter_list.[7]

  • return_type: Defines the data type of the value the function will send back to the caller. If a function does not return any value, its return_type should be void.[7]
  • function_name: A unique identifier for the function.
  • parameter_list: A comma-separated list of placeholder variables (parameters) that receive values (arguments) passed into the function when it is called.[7]

The general syntax for a function definition is: return_type function_name(parameter_list) { // Function body }.[7]

Function Call: To execute a function, its name is simply used, followed by parentheses, providing any necessary arguments: function_name(arguments);.[7] When a function is called, program control transfers to its body, and the statements within are executed.[7]

Return Values: The return keyword is used to send a single value back from the function to the point where it was called.[7] The type of this returned value must match the return_type declared in the function definition.

Parameters vs. Arguments: It is important to distinguish between "parameters" and "arguments." "Parameters" are the variables declared in the function's definition, acting as placeholders for incoming values. "Arguments" are the actual values passed during the function call.[7] The number, types, and order of arguments passed must precisely match the parameters defined in the function's signature.[7]

An example of a function that takes parameters and prints values:

#include <stdio.h> void printVal(int num, float real) { printf("%d %f\n", num, real); } int main() { int a = 3; printVal(a, 1.5); // Calling the function with arguments return 0; } // Output: 3 1.500000

When printVal(a, 1.5) is called, a (which is 3) is passed to num, and 1.5 is passed to real.[7]

In the context of the Plurality assignment, these function concepts are directly applied to the vote(string name) and print_winner() functions that require implementation. The vote function has a bool return type to indicate the success or failure of a vote, and it accepts a string name as a parameter, representing the candidate voted for. The print_winner function has a void return type, as it simply prints output to the console and does not need to return a value, and it takes no parameters. These functions serve as the specific locations where the logic developed using loops, conditionals, and structure access will be implemented.

CS50 places a strong emphasis on breaking down problems into smaller, manageable pieces, and functions are the primary mechanism for achieving this modularity.[7] The Plurality problem, while seemingly straightforward, involves distinct logical steps: processing command-line arguments, obtaining the voter count, handling individual votes, and finally determining and printing the winners. If all this logic were confined to the main function, it would become overwhelming for a beginner. The fact that CS50 provides the function prototypes for vote and print_winner is an implicit instruction that modularity is an expected and beneficial programming practice. This approach teaches a programmer not just *how* to write functions, but *why* they are essential for tackling larger, more complex programming problems, thereby instilling good software engineering habits from the outset.

A common source of errors for new programmers is misunderstanding how data flows into and out of functions, or failing to match function signatures precisely. The vote function explicitly defines a string name parameter and a bool return type. If a programmer attempts to call vote without a string argument, or tries to return an int instead of a bool, the compiler will generate errors. Similarly, if the name parameter is not correctly used inside the function, the logic will fail. Understanding this "contract" of a function – the precise types and order of inputs it expects and the type of output it guarantees – is crucial for writing correct, bug-free code.[7] This reinforces the importance of precise function interfaces for predictable and reliable program behavior.

Putting It All Together: Solving Plurality

This section provides step-by-step guidance for implementing the vote and print_winner functions in Plurality. We'll synthesize the concepts from the micro-lessons to build these functions logically. Follow the steps to understand the thought process.

Implementing the vote Function

The vote function (bool vote(string name)) updates a candidate's vote if name matches, returning true for success, false otherwise.

1. Iterate through Candidates: Use a for loop to go through each candidate in the global candidates array. Iterate from 0 up to candidate_count. It is important to remember that candidate_count is a global variable already established by the CS50 framework.[1][4]
// Inside bool vote(string name) for (int i = 0; i < candidate_count; i++) { // ... next steps inside loop ... }
2. Compare Names (Crucial for Strings): Inside the loop, use strcmp() to compare the input name with candidates[i].name. If strcmp() returns 0, the names match. strcmp() returns 0 when the two strings being compared are identical.[4]
// Inside the for loop if (strcmp(candidates[i].name, name) == 0) { // ... names match, next steps ... }
3. Update Votes & Return Success: If names match, increment candidates[i].votes and immediately return true;. This statement exits the function, signaling a successful vote and preventing further execution within vote.[4]
// Inside the if (strcmp(...) == 0) block candidates[i].votes++; return true; // Vote successful, exit function
4. Handle Invalid Votes (After Checking All Candidates): If the for loop completes its execution without finding any matching candidate (meaning the return true; statement was never reached), it implies that the name provided was invalid. Therefore, *after* the for loop, return false; should be executed to indicate an invalid ballot.[4]
// After the for loop concludes return false; // No matching candidate found

Common Pitfall: Avoid placing else { return false; } *inside* the loop, as it would make the function return after checking only the first non-matching candidate.

Implementing the print_winner Function

The print_winner function (void print_winner(void)) prints the name(s) of candidate(s) with the most votes.

1. Find the Maximum Vote Count (First Pass):
  • Initialize an integer variable, for example, max_votes, to 0 (or to the vote count of the first candidate, assuming there is at least one).
  • Use a for loop to iterate through the candidates array.
  • Inside this loop, use an if statement to compare the current candidate's votes (candidates[i].votes) with the max_votes found so far. If candidates[i].votes is greater than max_votes, update max_votes = candidates[i].votes;.[4]
// Inside void print_winner(void) int max_votes = 0; // Assuming candidate_count > 0, could initialize with candidates[0].votes // Or, more robustly, if candidate_count > 0, max_votes = candidates[0].votes; // then loop from i = 1. For simplicity here, starting with 0. for (int i = 0; i < candidate_count; i++) { if (candidates[i].votes > max_votes) { max_votes = candidates[i].votes; } }
2. Print All Candidates with Maximum Votes (Second Pass):
  • Once the first loop has completed, the max_votes variable will hold the highest vote count achieved by any candidate.
  • Now, use a *second* for loop to iterate through the candidates array *again*.
  • Inside this second loop, use an if statement to check if the current candidate's vote count (candidates[i].votes) is *equal* to max_votes.
  • If this condition is true, print the candidate's name followed by a newline character: printf("%s\n", candidates[i].name);.
// Still inside void print_winner(void), after finding max_votes for (int i = 0; i < candidate_count; i++) { if (candidates[i].votes == max_votes) { printf("%s\n", candidates[i].name); } }

Why two passes? You need to know the absolute maximum number of votes before you can identify all candidates who achieved that maximum, especially to handle ties correctly.[4]
